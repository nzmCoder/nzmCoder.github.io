Instructions for installing Microsoft Visual Studio C++ 6 on Windows 7

Yes, this tells how to install the 1998 version of Microsoft's C++ compiler on the modern Windows 7 OS.  I'll leave unanswered the questions of "why" you might want to do this, because there are two kinds of people in the world:  The people who already know why, and the people who will never understand. :-)

The only incompatibility I have seen is the inability to change fonts inside the program.  There is a workaround by making the adjustment using RegEdit.  Other than that it works just like it did in 1998, except that on a modern PC it starts up in 2 seconds, shuts down in 1 second, and compiles the average small program in about 3 seconds flat.  (I said I wasn't going to explain why anyone would still want to use this vesion, but there's a hint.)

Therefore, here is how to install it:

  1. You must be logged in as an admin with UAC (User Account Control) turned off. (This only needs to be during installation.)
  2. Create a zero length file named msjava.dll in the location: c:\windows (This makes the installer satisfied that the correct legacy version of java is already installed.)
  3. Browse into the install CD and run SETUP.exe
  4. A warning box regarding Visual Studio 6 being no good for Windows 7 will appear - Check box "Do not show this message again", and select Run Program. This applies to every such warning that occurs for the rest of this procedure.
  5. Enter the license key.  Click Next.
  6. Go into custom install.  UNcheck these items:
    • Microsoft Visual Foxpro
    • Microsoft Visual Interdev 6
    • Microsoft Visual Basic
    • Leave all other custom options as they were.
  7. On the Register Environment Variables query, leave box UNchecked.
  8. When prompted by message box, let the installer restart windows.
  9. After the reboot, the installer should pick up and continue by itself.
  10. Decline when queried to install Visual Sourcesafe, MSDN, InstallShield or anything else. Decline to register the product and let the install finish.
  11. Download and install Service Pack 6 for Visual Studio, available from Microsoft.
  12. Run the batch file, FixPermissions.bat . (This changes the compatibility properties of the executables.)
  13. Test the installation by running it, creating a project, and compiling.
  14. Log out and log back in as a user (not admin).
  15. Test the installation again as a user by running it, creating a project, and compiling.
